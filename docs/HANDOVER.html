The platform in one line

A candidate speaks to an AI voice examiner (real-time), is marked against each exam's own Jotform criteria, and receives an emailed PDF report. PassGP curates cases, exams and marking from an admin console.

Flow: the candidate's browser asks our backend for a short-lived token, then streams voice directly to OpenAI (the real key never leaves the server). When the exam ends, the backend grades the transcript against the case's marking criteria, saves the session to Supabase, and the report page emails a PDF.

Environment variables

Backend (Vercel → backend project)

OPENAI_API_KEY · SUPABASE_URL · SUPABASE_SECRET_KEY · SUPABASE_PUBLISHABLE_KEY · JOTFORM_API_KEY · Owner_Email · App_Password · ALLOWED_ORIGINS · ANALYTICS_API_KEY · (SMTP_HOST optional)

Admin & Chatbot (Vercel → each project)

VITE_SUPABASE_URL · VITE_SUPABASE_ANON_KEY · VITE_API_BASE

Secret keys live only on the backend / in Vercel — never in the browser apps and never committed to GitHub (.env is git-ignored).

Access & accounts

  • Admin login: admin@passgp.au (Supabase Auth)
  • Forgot password? Use “Reset password” on the admin login — a secure link is emailed to that inbox.
  • GitHub repo: Waqas56jb/ai-oral-examiner-mvp — pushing to main auto-deploys all 3 apps on Vercel.
  • Embed a case on Kajabi: the chatbot widget with ?caseId=, ?exam= or ?mock=N.

Please don't break these 🙏

  • Don't rename or remove the Vercel environment variables — the apps stop working without them.
  • Keep VITE_API_BASE pointing at the backend URL on the admin & chatbot projects.
  • Never paste secret keys into the browser apps or commit them to GitHub.
  • Case-bank edits go through the admin (the database blocks direct public writes by design).
  • Changing the AI's behaviour? Do it in Admin → AI Configuration / Exam Profiles first — no code change needed.
Where to change things without touching code: exam personalities, mark schemes & pass standards → Admin → Exam Profiles; voice, models & difficulty → Admin → AI Configuration; which cases the examiner uses → Admin → Training Panel; email & keys → Admin → Settings.
